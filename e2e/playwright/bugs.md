## BUG-AGT-004: Setting change not pushed live to connected agent

| Field | Value |
|-------|-------|
| **Affected setting** | `preferredExecutor` |
| **Expected** | Kaiad sends an updated frame to the already-connected agent WebSocket within 3 s of the operator saving preferredExecutor = cursor |
| **Observed** | No frame matching the updated value arrived on the existing WebSocket. The agent must disconnect and reconnect to receive the new setting. |

### Root cause

The Kaiad API server persists the updated tenant settings to the store
when an operator saves changes via the UI, but does **not** push those
changes to agent WebSocket sessions that are already connected. The Go
agent's `hello` frame is only sent once — at connection time — so a
running agent never learns about setting changes until it reconnects.

### Reproduction steps

1. Start Kaiad and enroll a Go agent.
2. While the agent is connected, navigate to **Tenants → Configure**.
3. Change **preferredExecutor** and click **Save tenant settings**.
4. Observe: the connected agent does not receive any updated message.

### Acceptance criteria

After saving, Kaiad should send an updated `hello` (or a new
`settings_update` frame) to every WebSocket session belonging to the
affected tenant so agents update without requiring a reconnect.
